# XDE-001–010 — Executive Documentary Engine Canon

**Status:** Canonical
**Authority:** Approved by Darren Cureton
**Effective:** Immediately
**Applies to:** XEI executive premieres, CEO demonstrations, board experiences, strategic showcases, investor experiences, flagship product premieres, and all derivative executive media.

## Governing premise

An executive experience is a directed documentary that happens to be interactive. It is not a slide deck, feature tour, dashboard walkthrough, or software demonstration. Its job is to move an executive from curiosity to recognition, ownership, vision, and mandate while preserving technical reliability.

The approved nine-scene Ed Executive Premiere is the current production skeleton. Future work evolves the skeleton incrementally. Replacing it requires explicit approval.

## XDE-001 — Executive Documentary Engine

Every experience is story-first, emotion-driven, visually memorable, executive-focused, and technically dependable. The canonical emotional progression is:

Arrival → Curiosity → Recognition → Tension → Proof → Participation → Ownership → Scale → Vision → Mandate.

Every scene must answer one question, reveal one idea, create one visual memory, change the executive’s mental state, and create the next question.

## XDE-002 — Documentary Production Pipeline

Every scene moves through four controlled stages:

1. **Narrative:** question, reveal, emotional shift, next question.
2. **Storyboard:** camera, composition, hero object, lighting, motion, transition.
3. **Experience:** interaction, timing, silence, narration, captions, progression.
4. **Engineering:** faithful implementation of the approved storyboard.

Engineering may not invent story architecture during implementation.

## XDE-003 — Living Executive Language

Use human executive language rather than systems language.

- “Knowledge propagates” becomes “Your company just learned something.”
- “Executive memory” becomes “Nothing important gets forgotten again.”
- “Knowledge graph” becomes “Every decision has somewhere to live.”

Xen speaks less. Silence is intentional. One precise sentence is preferred to a paragraph.

## XDE-004 — Visual Hierarchy

Not every scene carries equal intensity. Experiences require controlled visual peaks. The finale must feel materially larger than the opening. Each scene has one hero object and one dominant visual idea.

## XDE-005 — Documentary Music and Sound

Sound follows emotional movement: arrival, wonder, tension, discovery, ownership, scale, vision, silence. Music never covers comprehension or becomes a runtime dependency. Silent operation must remain fully coherent.

## XDE-006 — Executive Participation

The executive should not remain passive for more than approximately 60–90 seconds. Participation follows the rhythm:

Watch → Think → Touch → Watch → Decide → Watch → Contribute → Watch → Approve.

Every interaction must alter the story, retain meaning, or influence the final mandate.

## XDE-007 — WOW Ledger

Each scene receives exactly one memorable “WOW” event. Multiple competing WOW events dilute impact. The production record must identify each scene’s single visual memory.

## XDE-008 — Five-Second Rule

The core idea of every scene must be understandable within five seconds without narration. Headlines, composition, and the hero visual must communicate before detailed reading.

## XDE-009 — Documentary Budget

Every scene is limited to:

- one hero animation,
- one interaction,
- one reveal,
- one transition,
- one visual metaphor,
- one emotional objective.

Additional complexity requires an explicit exception and a demonstrated reliability benefit.

## XDE-010 — Living Company Signature Moments

Executive documentaries should draw from five canonical signature moments when relevant:

1. **Knowledge Loss:** a person leaves; hard-earned judgment disappears.
2. **Memory:** a decision is made once and retained.
3. **Learning:** approved judgment changes the operating system.
4. **Scale:** one company becomes the proof for the group.
5. **Legacy:** the organization remembers beyond current leadership.

## Executive Experience Review Board (EERB)

The EERB is now part of the overall Xen review board. Engineering approval alone is insufficient. A release must pass:

1. Narrative Continuity
2. Emotional Progression
3. Executive Clarity
4. Visual Memory
5. Technical Reliability
6. Mandate Creation

Each gate is scored PASS, CONDITIONAL, or FAIL with evidence.

## Experience Stability Principle — ESP-001

Once a structural foundation has passed user testing, innovation occurs inside that foundation whenever practical. Do not restart, duplicate controllers, or stack conflicting behavior. Preserve working paths and improve one authoritative owner.

## Current production inheritance

The Ed Executive Premiere currently implements:

- one synchronous entry action,
- one navigation owner,
- nine scene progression,
- curiosity-based action labels,
- one Ed participation moment,
- one visible propagation moment,
- one enterprise scale moment,
- one mandate decision,
- one cinematic close,
- phone-safe viewport behavior,
- silent coherent operation,
- no inherited voice or director dependency.

## Release rule

Narrative approval → storyboard approval → experience implementation → runtime validation → EERB certification → merge → live-route verification → physical-device review.

Static string checks do not prove physical-device acceptance. Repository certification proves implementation contract only; Darren’s phone review remains the final experience acceptance gate.
